# orval-demo

Demo project for exploring [orval](https://orval.dev/) — TypeScript client codegen from OpenAPI spec.

## Stack

| Layer | Technology |
|-------|-----------|
| Monorepo | pnpm workspaces |
| Backend | [Hono](https://hono.dev/) + [@hono/zod-openapi](https://github.com/honojs/middleware/tree/main/packages/zod-openapi) |
| Schema validation | [Zod](https://zod.dev/) |
| Codegen | [orval](https://orval.dev/) |
| Frontend | [Angular 22](https://angular.dev/) |
| Frontend architecture | [Feature-Sliced Design](https://feature-sliced.design/) |

## How it works

```
Hono + @hono/zod-openapi
        ↓ generates
  /openapi.json
        ↓ orval reads
  Angular services + TypeScript types (src/shared/api/)
```

## Structure

```
apps/
├── backend/
│   └── src/
│       ├── schemas/todo.ts        ← Zod schemas (Todo, CreateTodo, NotFound)
│       ├── routes/todos.ts        ← CRUD routes /todos
│       └── index.ts               ← Hono app + server
└── frontend/
    ├── src/
    │   ├── shared/api/            ← generated by orval (do not edit manually)
    │   ├── entities/todo/         ← TodoCard component
    │   ├── features/
    │   │   ├── create-todo/       ← CreateTodoForm component
    │   │   └── delete-todo/       ← DeleteTodoButton component
    │   ├── widgets/todo-list/     ← TodoList component
    │   └── pages/todos-page/      ← state + orchestration
    └── orval.config.ts            ← codegen configuration
```

### FSD import rules

Each layer can only import from layers below it:

```
pages → widgets → features → entities → shared
```

Path aliases (configured in `tsconfig.json`):

```ts
import { Todo, DefaultService } from '@shared/api';
import { TodoCardComponent } from '@entities/todo';
import { CreateTodoFormComponent } from '@features/create-todo';
import { TodoListComponent } from '@widgets/todo-list';
import { TodosPageComponent } from '@pages/todos-page';
```

## Requirements

- Node.js `>=22.22.3` ([nvm](https://github.com/nvm-sh/nvm) recommended)
- pnpm `>=11`

```bash
nvm use 22
npm install -g pnpm
```

## Installation

```bash
pnpm install
pnpm approve-builds --all
```

## Running

### 1. Backend

```bash
pnpm dev:backend
# http://localhost:3000
# http://localhost:3000/openapi.json
```

### 2. Codegen (while backend is running)

```bash
pnpm codegen
# generates apps/frontend/src/shared/api/
```

### 3. Frontend

```bash
pnpm dev:frontend
# http://localhost:4200
```

## API

| Method | Path | Description |
|--------|------|-------------|
| GET | `/todos` | List todos |
| GET | `/todos/:id` | Get todo by id |
| POST | `/todos` | Create todo |
| DELETE | `/todos/:id` | Delete todo |

Paste `/openapi.json` contents into [editor.swagger.io](https://editor.swagger.io) for Swagger UI.

## Codegen

`apps/frontend/orval.config.ts` reads the spec from the running backend and generates Angular `HttpClient` services into `src/shared/api/`.

After changing backend schemas: restart backend → `pnpm codegen`.
